# CM2Editer 国际化（i18n）设计方案

> 版本：v3.0  
> 状态：阶段三已完成（节点元数据 i18n 化，英文翻译为基础版）  
> 首批语言：简体中文（zh）、英语（en）  
> 预留语言：日语（ja），暂由英语 fallback 占位

---

## 1. 背景与目标

当前编辑器所有 UI 文本（节点显示名、描述、端口名、参数名、按钮、状态消息、tooltip 等）均为硬编码中文。这导致：

- 非中文用户无法使用编辑器。
- 后续按使用场景重构节点面板时，新增分类名称、悬停提示等文本会进一步固化中文，增加后续迁移成本。
- 无法支持日本发售地及国际英语社区。

本方案目标是建立轻量、运行时切换的多语言机制，首批支持 **简体中文（zh）** 和 **英语（en）**，并预留 **日语（ja）** 扩展位置。

---

## 2. 语言策略

| 语言 | 代码 | 首批实现 | 说明 |
|------|------|----------|------|
| 简体中文 | `zh` | ✅ 主维护语言 | 开发者母语，准确可控 |
| 英语 | `en` | ✅ 首批 | 国际社区通用语言 |
| 日语 | `ja` | ⏸ 预留 / 占位 | 发售地为日本，但需母语者贡献高质量翻译；缺失时 fallback 到英语 |

**不首批实现日语的原因**：

- 中文与日语虽共享部分汉字，但现代 UI 文本差异显著（如“装备”→“装備”、“高潮”→“絶頂”），不能依赖“东亚相似”节省翻译。
- 非母语翻译会损害日本用户体验，宁可使用英语 fallback。

---

## 3. 技术架构

### 3.1 核心结构

新增 `src/ui/i18n.rs`（或 `src/i18n.rs`）：

```rust
pub struct I18n {
    /// 当前语言代码，如 "zh"、"en"。
    current: String,
    /// 语言代码 -> (键 -> 文本) 的映射。
    translations: HashMap<String, HashMap<String, String>>,
}

impl I18n {
    pub fn new() -> Self;
    pub fn load_from_dir(&mut self, dir: &Path) -> Result<()>;
    pub fn set_language(&mut self, lang: &str);
    pub fn current_language(&self) -> &str;
    pub fn text(&self, key: &str) -> String;
    pub fn text_opt(&self, key: &str) -> Option<String>;
}
```

### 3.2 翻译文件

放置于 `assets/i18n/`：

```text
assets/i18n/
├── zh.json
├── en.json
└── ja.json   // 首批仅作为占位/预留
```

文件格式为 JSON 键值对：

```json
{
    "panel.project": "工程",
    "panel.namespace": "命名空间",
    "button.save": "保存",
    "status.saved": "已保存工程 {}",
    "node.goto.name": "跳转",
    "node.goto.description": "跳转到指定标签"
}
```

### 3.3 Fallback 机制

1. 优先查找当前语言的 key。
2. 缺失时查找英语（`en`）。
3. 仍缺失则返回 key 本身（便于调试定位）。

### 3.4 与 egui 结合

- 在 `App` 中持有 `pub i18n: I18n`。
- 各面板函数增加 `i18n: &I18n` 参数（如 `NodeLibraryPanel::show(ui, i18n, ...)`）。
- 语言切换入口放在设置菜单中，切换后即时生效。

## 4. 键命名规范

使用点分命名，按命名空间组织，避免冲突。

| 命名空间 | 示例 | 说明 |
|----------|------|------|
| `app.*` | `app.title` | 应用级文本 |
| `panel.*` | `panel.project`, `panel.namespace` | 左侧面板标题 |
| `button.*` | `button.save`, `button.export` | 按钮文本 |
| `status.*` | `status.saved` | 底部状态消息 |
| `node.{type}.name` | `node.goto.name` | 节点显示名 |
| `node.{type}.description` | `node.goto.description` | 节点描述 |
| `node.{type}.param.{name}` | `node.goto.param.label` | 节点参数显示名 |
| `node.{type}.port.{id}` | `node.goto.port.out_label` | 节点端口显示名 |
| `port.*` | `port.in_flow`, `port.out_flow` | 通用端口名 |
| `param.*` | `param.value`, `param.condition` | 通用参数名 |
| `tooltip.*` | `tooltip.coord_picker` | 悬停提示 |
| `dialog.*` | `dialog.new_project.title` | 对话框文本 |
| `catalog.*` | `catalog.task_flow`, `catalog.condition` | 节点面板场景分类（预留） |
| `error.*` | `error.no_project` | 错误提示 |

`{type}` 使用 `NodeType` 的 `PascalCase` 名称，如 `Goto`、`If`、`EquipCosplay`、`CheckCosplay`。

## 5. 与现有模块的边界

### 5.1 保持 `api/definitions.rs` 不变

`NodeDefinition` 中当前硬编码中文显示名、描述、端口名、参数名。首批不改动该结构，避免影响序列化与代码生成。

### 5.2 渲染时动态替换

在 `api/registry.rs` 或 `ui/i18n.rs` 中维护 `NodeType → i18n key` 映射。渲染节点库、属性面板时，通过 `i18n.text(key)` 获取翻译文本。

```rust
pub fn node_display_name(i18n: &I18n, node_type: NodeType) -> String {
    let key = format!("node.{:?}.name", node_type);
    i18n.text(&key)
}
```

未来若需要，`NodeDefinition` 中的硬编码字段可逐步迁移为 i18n key 或完全移除。

### 5.3 代码生成不受 i18n 影响

`.code` 生成器只使用 `NodeType` 枚举和 `ParamValue`，不依赖显示文本，因此 i18n 完全局限于 UI 层。

## 6. 使用示例

### 6.1 普通 UI 文本

```rust
ui.heading(i18n.text("panel.properties"));
if ui.button(i18n.text("button.save")).clicked() { ... }
```

### 6.2 带格式参数

```rust
let msg = i18n.text("status.saved").replace("{}", &project_name);
self.status_message = msg;
```

或扩展 `I18n` 提供 `format` 方法：

```rust
pub fn format(&self, key: &str, args: &[&str]) -> String {
    let mut s = self.text(key);
    for arg in args {
        s = s.replacen("{}", arg, 1);
    }
    s
}
```

### 6.3 节点显示

```rust
let name = i18n.text(&format!("node.{:?}.name", node_type));
let desc = i18n.text(&format!("node.{:?}.description", node_type));
ui.heading(name);
ui.label(desc);
```

## 8. 语言持久化与翻译文件加载路径

- 新增 `src/settings.rs`：`AppSettings` 保存 UI 语言到用户配置目录（Windows 下为 `%APPDATA%/CM2Editer/settings.json`）。
- 应用启动时加载持久化的语言偏好；语言选择器切换时自动保存。
- `I18n::load_bundled` 增加加载路径回退：依次尝试当前工作目录、`target/debug/` 同级目录、可执行文件目录，确保 `cargo run`、IDE 调试、打包运行都能找到 `assets/i18n/`。

## 9. 与节点面板场景分类的关系

节点面板场景分类（任务/流程、条件判定、数据获取等）将直接使用 i18n 的 `catalog.*` 命名空间：

```json
{
    "catalog.task_flow": "任务 / 流程",
    "catalog.task_flow.mission": "任务面板",
    "catalog.task_flow.threading": "线程与监听器",
    "catalog.condition": "条件判定",
    "catalog.data_read": "数据获取",
    "catalog.data_write": "数据修改",
    "catalog.data_process": "数据处理"
}
```

这样可以先完成 i18n 基础设施，再无痛实施分类重构。

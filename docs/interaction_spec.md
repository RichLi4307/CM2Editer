## 一、画布状态机

编辑器画布只有以下 6 种状态，任何时刻必须处于且仅处于一种状态：

| 状态 | 触发条件 | 允许的操作 |
|------|---------|-----------|
| `Idle` | 默认状态 | 左键点击选中、中键拖拽平移、滚轮缩放、双击空白呼出搜索、按 Space 呼出搜索 |
| `Panning` | 中键按下 | 鼠标移动 → 画布平移；中键释放 → 回到 Idle |
| `DraggingNode` | 左键在节点上按下 | 鼠标移动 → 节点跟随；释放 → 放置，回到 Idle |
| `DrawingEdge` | 左键在端口上按下并拖拽 | 鼠标移动 → 绘制临时连线；释放到兼容端口 → 创建边；释放到空白 → 取消 |
| `BoxSelecting` | 左键在空白处拖拽（移动超过 4px 阈值后触发） | 鼠标移动 → 绘制框选矩形；释放 → 选中框内节点，回到 Idle |
| `ContextMenu` | 右键点击节点/边/空白 | 显示菜单；点击外部 → 关闭，回到 Idle |

### 状态转移约束

- `DrawingEdge` 状态下，滚轮仍可用（缩放），但节点不可拖拽、不可选中
- `Panning` 状态下，节点不可被选中、不可拖拽
- `BoxSelecting` 状态下：
  - 按住 **Shift** 释放 → 框内节点追加到当前选集（Extend）
  - 按住 **Ctrl** 释放 → 框内节点从当前选集中移除（Subtract）
  - 无修饰键释放 → 框内节点替换当前选集（Set）
  - 框选模式（Window/Crossing）始终由拖拽方向决定，不受修饰键影响
- 任何状态下，Ctrl+Z / Ctrl+Y / Ctrl+S / Delete 始终响应（全局快捷键）
- 状态互斥：`ContextMenu` 打开期间，所有其他交互暂停

## 二、输入映射速查表

| 输入 | 作用域 | 行为 |
|------|--------|------|
| 左键单击 | 节点 | 选中该节点，取消其他选中 |
| 左键按住拖拽 | 节点 | 开始拖拽（触发 `DraggingNode`） |
| 左键按住拖拽 | 端口 | 开始拖线（移出端口 4px 阈值后触发 `DrawingEdge`） |
| 左键单击 | 空白处 | 取消所有选中（按下-释放位移 < 4px 时视为单击，而非框选） |
| 左键拖拽 | 空白处（移动 ≥ 4px） | 进入 `BoxSelecting`；方向决定模式：左→右 = Window（严格包含），右→左 = Crossing（触碰交叉） |
| Shift + 左键拖拽 | 空白处 | 追加框选（框内节点加入已有选集，不取消框外选中） |
| Ctrl + 左键拖拽 | 空白处 | 减选框选（框内节点从已有选集中移除） |
| Ctrl + 左键单击 | 节点 | 切换该节点选中状态（多选追加） |
| 中键按住拖拽 | 任意 | 平移画布（`Panning`） |
| 右键单击 | 节点 | 节点上下文菜单 |
| 右键单击 | 边 | 边上下文菜单（删除、翻转方向） |
| 右键单击 | 空白 | 画布上下文菜单（新建节点、粘贴） |
| 滚轮 | 任意 | 缩放，以鼠标指针为中心，范围 0.1x ~ 4.0x |
| Space | 全局 | 呼出快速搜索面板（创建节点） |
| Delete | 全局 | 删除选中节点/边 |
| Ctrl+Z | 全局 | 撤销（至少 50 步） |
| Ctrl+Y | 全局 | 重做 |
| Ctrl+S | 全局 | 保存 JSON 文件 |
| Ctrl+C | 全局 | 复制选中节点 |
| Ctrl+V | 全局 | 粘贴节点（位置偏移 +40, +40） |
| Ctrl+D | 全局 | 原地克隆选中节点 |

**拖拽阈值**：空白处左键按下后，鼠标移动距离 ≥ 4px 才触发 `BoxSelecting`，否则视为单击（取消选中）。阈值判定以 `mousedown` 位置为原点，使用欧几里得距离。

## 三、拖线实时反馈

用户从端口拖出连线时，必须实时反馈目标端口的可连接性。

### 3.1 拖拽过程中的视觉状态

| 条件 | 目标端口表现 | 临时连线颜色 |
|------|-------------|-------------|
| 类型兼容，无环风险 | 端口放大 + 白色外圈 2px | Flow 边=白色，Data 边=对应端口类型颜色 |
| 类型不兼容 | 端口变灰 | 红色虚线 |
| 类型兼容但会形成 DAG 环 | 端口变红 + 抖动 1px | 红色实线 |
| Data 端口已有一条入边 | 端口变橙 | 橙色虚线 |
| 自环（`from.node == to.node`） | 端口变红 | 红色实线 |
| 鼠标悬停在空白处 | 无目标端口 | 连线末端为虚线圆点 |

### 3.2 释放时的行为

| 释放位置 | 结果 |
|---------|------|
| 合法目标端口 | 创建边，进入 Undo 栈，回到 Idle |
| 非法目标端口（红/橙/灰） | 不创建边，临时连线 200ms 淡出，回到 Idle |
| 空白处 | 不创建边，临时连线立即消失，回到 Idle |
| 源端口自身 | 不创建边，无淡出，回到 Idle |

### 3.3 环检测算法

在 `DrawingEdge` 状态下，每帧执行：

1. 获取当前鼠标位置下最近的端口
2. 若目标端口存在且类型兼容：
   - 暂存当前边到 Graph 的副本，运行 `src/graph/validation.rs` 中的 `check_no_cycles`（Kahn 算法）
   - 若检测到环 → 标记为"环风险"（红色反馈）
3. 若目标端口是 Data 输入端口且已有入边 → 标记为"已占用"（橙色反馈）
4. 若目标端口与源端口属于同一节点 → 标记为"自环"（红色反馈）

> 性能说明：节点数 < 100 时 Kahn 算法每帧开销可忽略。若未来节点数 > 500，可将检测频率降为每 100ms 一次。

### 3.4 框选矩形样式

| 拖拽方向 | 框选模式 | 矩形边框 | 内部填充 | 语义 |
|---------|---------|---------|---------|------|
| 左 → 右 | Window（严格包含） | 蓝色实线 2px（`#2196F3`） | 半透明蓝（`#2196F3`，alpha 0.15） | 只有完全在框内的节点才被选中 |
| 右 → 左 | Crossing（触碰交叉） | 绿色虚线 2px（`#4CAF50`，dash 5px / gap 3px） | 半透明绿（`#4CAF50`，alpha 0.15） | 只要节点与框边界相交即被选中 |

### 3.5 候选高亮

框选拖拽过程中，被框选矩形覆盖的节点实时进入**候选状态**：

- 候选节点边框显示 50% 透明度的高亮（与最终选中状态的蓝色发光区分）
- 释放鼠标后，候选节点根据当前修饰键转为正式选中/追加/减选

### 3.6 空框行为

若释放时框选矩形未覆盖任何节点：

- 框选矩形执行 150ms 收缩消失动画
- 无节点选中状态变更（无操作进入 Undo 栈）

## 四、空画布启动体验

### 4.1 首次打开 / 新建项目

当 `Graph` 的 `nodes` 为空时：

1. 画布中央自动创建一个 `Start` 节点
2. 节点 `position` 设为 `{ x: 0, y: 0 }`
3. 该节点自动进入**选中状态**
4. 右侧属性面板自动展开，显示提示文本："任务入口节点"
5. 左栏节点库默认展开「Control」分类

### 4.2 删除最后一个节点后

当用户删除节点导致 `nodes` 为空时：

1. 画布中央显示半透明提示文本：`"按 Space 添加节点"`
2. **不自动创建** Start 节点（避免误删后强制重建）
3. 按 Space 呼出搜索面板，选中节点后自动放置于画布中心 `{ x: 0, y: 0 }`

## 五、撤销/重做粒度

以下操作各占用 Undo 栈的一个独立步骤：

| 操作 | Undo 行为 |
|------|----------|
| 移动节点（拖拽或框选移动） | 回退到移动前的位置 |
| 创建节点 | 删除该节点及其连接的边 |
| 删除节点 | 恢复节点及其全部连接的边 |
| 创建边 | 删除该边 |
| 删除边 | 恢复该边 |
| 修改节点参数（每个字段） | 回退到修改前的值 |
| 折叠/展开节点 | 切换折叠状态 |
| 粘贴节点 | 删除粘贴产生的节点和边 |

### 合并规则

- 连续移动同一节点，间隔 < 500ms → 合并为一次 Undo 步骤
- 框选移动多个节点 → 视为一次 Undo 步骤
- 参数编辑：同一字段连续输入，在失焦或 Enter 确认后 → 视为一次 Undo 步骤

### 栈容量

- 至少 50 步，超出时丢弃最早步骤
- 新建项目 / 加载文件时清空 Undo 栈

## 六、节点选中与多选

| 操作 | 结果 |
|------|------|
| 左键单击节点 | 选中该节点，取消其他所有选中 |
| Ctrl + 左键单击节点 | 切换该节点选中状态（追加/移除多选） |
| 框选 | 选中框内全部节点，取消框外选中 |
| Shift + 框选 | 追加框内节点到当前选中集合 |
| 左键单击空白 | 取消所有选中 |
| 左键拖拽空白（左→右） | Window 框选：选中完全包含在矩形内的节点，取消其他选中 |
| 左键拖拽空白（右→左） | Crossing 框选：选中与矩形边界相交的节点，取消其他选中 |
| Shift + 左键拖拽空白 | 追加框选：保留已有选集，将框内节点加入 |
| Ctrl + 左键拖拽空白 | 减选框选：保留已有选集，将框内节点移除 |

选中节点时：

- 节点边框：2px 蓝色发光（`#2196F3`，alpha 0.8）
- 右侧属性面板显示**最后选中节点**的参数表单
- 多选时拖拽任一选中节点，所有选中节点一同移动

## 七、错误节点高亮

验证器（`src/graph/validation.rs`）发现错误的节点在画布上的展示：

- 节点边框变为红色（`#F44336`，2px 实线）
- 鼠标悬停时显示 Tooltip，列出该节点的所有验证错误信息
- 错误不阻止编辑操作，但保存 JSON / 导出 `.code` 时弹出确认对话框
- 错误消失（节点被修补后）立即恢复正常边框

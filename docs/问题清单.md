## 未解决问题

### 1. If 节点 condition 无法通过 DataFlow 连接

**优先级**：P1
**症状**：当前 If 节点的 `condition` 为布尔类型参数，只能手动勾选 true/false，无法从其他节点的数据输出端口连接。
**处理**：归入 DataFlow 重构计划统一处理。

### 2. Log 节点参数清空后变为 null 触发验证错误

**优先级**：P2
**症状**：Log 节点 output 参数（String 类型）清空后，`parse_param_value` 中空字符串被解析为 `ParamValue::Null`，验证器 `check_required_params` 检测到 Null 值后报错。
**文件**：`src/ui/panels/properties.rs:111` parse_param_value
**建议方案**：必填 String 参数清空时应设为 `Literal("")` 而非 `Null`；或者验证器跳过默认值类型检查。

## 作者建议与展望

以下建议来自作者的三轮测试反馈，列为长期设计参考：

### 交互与视觉

- 连线风格建议改为**电路连接线**（非贝塞尔曲线）
- 鼠标靠近输出端口时放大**吸附环**，提升拖线操作感
- 节点折叠时高度自适应；未来支持 resize handle 手动调整

### 数据与 API

- General Function 节点参数类型模糊或错误，需系统审查 `docs/` 中 API 文档，统一 `ParamType` 类型
- 所有可枚举变量（如物品类型、场景、动作、技能等）应改为**下拉表**而非自由文本
- `selected_cosplay.json` 及其他命名空间 JSON，作为数据源提供**搜索 + 译名 + 命名空间**面板

### 架构建议

- **左栏二级菜单**：一级菜单分文件操作、命名空间管理、坐标菜单；节点库为二级子菜单
- **坐标"语言糖"**：预制坐标/视角变量，编辑器可视化 x/y/z，`.code` 导出时做语言转译。需评估与数学/变量节点的冲突风险
- **JSON 预览栏**：支持拖动调节高度，方便审查数据一致性
- **Graph 冲突检测**：例如 S→A→B 和 S→B 两个子图共享同一个 Start，当前无任何提示

### 编辑器定位反思

> 本质上是节点化的 `.code` 代码块编辑器。所有"语言糖"、预制变量、坐标管理等高级功能，都需要确保与现有数学/变量类型节点不产生语义冲突。建议产出我们自己的 **API 文档**，明确编辑器层面的概念模型（变量、坐标、视角等）与 `.code` 生成层的映射关系。

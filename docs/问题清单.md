## 未解决问题

### 1. 编辑器缺乏工程/项目管理能力（发布前必做）

**优先级**：P0 / 发布阻塞
**依据**：
- `docs/documentation_zh.html#File-Structure`：每个项目在游戏 `CustomMissions2` 文件夹中拥有自己的子文件夹。
- `docs/documentation_zh.html#Code`：项目可拆分为多个 `.code` 文件，加载时合并。
- `docs/documentation_zh.html#Meta`：项目必须包含 `meta.json`，定义多语言标题、描述、设置菜单（`_settings`）。
- `docs/examples/` 中所有示例（`Test`、`NPC_type`、`drop bra and panties`、`MessengerExample`）均为"文件夹 + `meta.json` + 多个 `.code` 文件"结构。
**当前问题**：
1. `export_code` 仅生成单个 `.code` 文件，不生成项目文件夹和 `meta.json`。
2. `save_json` 将编辑器内部状态保存为单个 `.json`，与真实项目结构脱节。
3. 没有内置 `.code` 文本编辑器。
4. 左栏没有工程文件树，无法管理多个 `.code` 文件和 `meta.json`。
5. 新建/保存工程时没有让用户选择项目文件夹并命名。
**解决方案**：引入 `Project` 概念；编辑器保存时同时生成/更新 `meta.json` 和所有 `.code`；左栏增加工程文件树；内置 `.code` 文本编辑器；新建工程时选择文件夹并命名。详见 `docs/TODO.md` Phase 4.5。

### 2. If 节点 condition 无法通过 DataFlow 连接

**优先级**：P1
**症状**：当前 If 节点的 `condition` 为布尔类型参数，只能手动勾选 true/false，无法从其他节点的数据输出端口连接。
**处理**：归入 DataFlow 重构计划统一处理。

### 2. Log 节点参数清空后变为 null 触发验证错误

**优先级**：P2
**症状**：Log 节点 output 参数（String 类型）清空后，`parse_param_value` 中空字符串被解析为 `ParamValue::Null`，验证器 `check_required_params` 检测到 Null 值后报错。
**文件**：`src/ui/panels/properties.rs:111` parse_param_value
**建议方案**：必填 String 参数清空时应在json端设为 `Literal("")` 而非 `Null`；或者验证器跳过默认值类型检查。

## 未解决问题

### 1. If 节点 condition 无法通过 DataFlow 连接

**优先级**：P1
**症状**：当前 If 节点的 `condition` 为布尔类型参数，只能手动勾选 true/false，无法从其他节点的数据输出端口连接。
**处理**：归入 DataFlow 重构计划统一处理。

### 2. Log 节点参数清空后变为 null 触发验证错误

**优先级**：P2
**症状**：Log 节点 output 参数（String 类型）清空后，`parse_param_value` 中空字符串被解析为 `ParamValue::Null`，验证器 `check_required_params` 检测到 Null 值后报错。
**文件**：`src/ui/panels/properties.rs:111` parse_param_value
**建议方案**：必填 String 参数清空时应设为 `Literal("")` 而非 `Null`；或者验证器跳过默认值类型检查。

### 来源：`问题清单-2026-07-08-03-28.md`

> log 节点填充数据后清空，会变成 null 并在底部显示 1 个错误
> General Function 类型的节点，数据参数类型模糊或错误，查阅 docs 中的 API 文档进行重构，可枚举的变量改成下拉表
> 我提供了 selected_cosplay.json 在 docs 文件夹，这个是 cosplay 服装参数的命名空间，考虑到内容太多，在节点涉及填入这些参数时，提供一个新窗口，可以选择、搜索、显示译名和命名空间。
> 左栏应分二级菜单，当前使用的节点表被归纳到节点菜单，一级菜单包括文件操作菜单入口、命名空间管理入口（振动器、活塞强度这些都是可枚举的，需要一个快速导入导出和复制的地方），格式建议匹配 selected_cosplay.json。然后是坐标菜单，可以快速管理坐标、拖出预制成坐标变量、视角方向的全局/局部变量
> 这里点名传送、坐标获取节点等。考虑到 .code 是一种编码语言，我们可以预制一些"语言糖"，比如坐标变量、视角方向，在编辑器里可以便捷更改 x、y、z 参数，但在生成 .code 时需要进行合适的语言转译。该想法需要仔细评估可行性。因为本质上是节点化的 .code 代码块编辑器，所以甚至有可能与数学、变量类型功能的节点冲突。这方面需要我们自己的 API 文档
> 给 json 预览栏变成可调高度的

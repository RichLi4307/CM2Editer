# CM2Editer 问题清单

> 更新时间：2026-07-07 20:41
> 优先级：P0 阻塞 → P1 核心缺失 → P2 严重体验 → P3 视觉/优化

---

## P0 — 阻塞可用性

### 导出 .code 生成空文件

**症状**：对话框和状态栏正常，但输出文件大小为 0 字节，无错误信息。
**文件**：`src/code_gen/generator.rs` → `generate_code_to_file`，对空图的逻辑或文件写入路径需排查。

---

## P1 — 核心体验缺失

### 节点内部参数无法编辑

**症状**：属性面板可显示参数名和值，但无法修改（Input/Select 等控件不可交互或值不回调写入）。这是编辑器最核心的功能之一。
**文件**：`src/ui/panels/properties.rs` → `PropertiesPanel::show` 的返回值处理 / `SetParam` 命令执行链。

### 端口介绍文字不渲染

**症状**：节点上仅显示标题，端口名称和参数预览不可见。疑似与折叠状态 `collapsed` 或图层/裁剪有关——节点 body 区域内容（端口标签、参数预览）未绘制。
**文件**：`src/ui/node_renderer.rs` → `paint_port` 的 label 绘制、`paint_param_preview` 可见性。

### 折叠/展开按钮无反应

**症状**：右键菜单点"折叠/展开"后节点外观不变。
**文件**：`src/ui/interaction.rs:show_context_menu` → 设置 `node.collapsed` 后 `node_renderer.render_with_data` 应跳过 `paint_param_preview`。检查是否因为旧 `layout_and_paint_ports` 仍绘制了所有端口说明。

---

## P2 — 严重影响使用

### Ctrl+C / Ctrl+V 快捷键失效

**症状**：已改用 `consume_key` 但两个快捷键仍不生效。可能原因：搜索框 `NodeLibraryPanel::show_search` 的 text_edit 消费了按键、或事件顺序问题（先画面板后快捷键被消费）。
**文件**：`src/app.rs:update` 快捷键处理 / `src/ui/panels/node_library.rs` 的 search 输入框。

### 粘贴后状态机未刷新

**症状**：粘贴新节点后，拖动重叠区域时旧节点被拖动（而非置顶的新节点）。鼠标移出再移回后恢复正常。疑似 `InteractionController` 的 `context_menu` 或节点命中 Rect 未在 paste 后刷新。
**文件**：`src/app.rs:paste_at` → `handle_input` 的节点命中计算 / `update_canvas` 的 `node_hits` 重建时机。

### 100 Start 节点帧率 <30 fps

**症状**：添加 100 个 Start 节点后帧率低于 30（视口裁剪已启用）。非 GPU 瓶颈（显卡功耗无变化），疑似 CPU 瓶颈：每帧遍历全部节点判断裁剪、文本排版、serde 序列化（JSON 预览面板）等。
**策略**：需 profiler 定位热点，候选方向：① JSON 预览懒更新/节流 ② 节点数据 Vec 只收集可见节点而非全部 nodes.values() ③ 文本缓存。

### "新建"未提示保存

**症状**：工具栏"新建"直接清空，未保存的数据丢失。
**文件**：`src/app.rs:new_graph` → 应在 `new_graph()` 前检查 `!graph.nodes.is_empty()` 并弹出确认。

## P3 — 视觉细节 / 优化

### "✕" 按钮渲染为方框

**症状**：搜索窗口关闭按钮 `✕` 当前字体不支持，显示为方框。
**修复**：改为字母 `X` 或文字 `Close`。

### 右键空白处粘贴按钮未置灰

**症状**：剪贴板为空时"粘贴"仍可点击。
**修复**：`ui.add_enabled(!clipboard.nodes.is_empty(), ...)` 包裹按钮。

### 部分节点参数与端口标签重叠

**症状**：节点内参数预览文字与端口标签文字位置冲突。
**文件**：`src/ui/node_renderer.rs` → `paint_param_preview` 的起始 y 偏移需要避开端口行。

### 加载损坏 JSON 时无弹窗

**症状**：JSON 解析失败仅在状态栏显示红色文字，用户可能未注意。
**建议**：可加 `rfd::MessageDialog` 或 egui modal 弹窗，非必须——状态栏已满足基本需求。

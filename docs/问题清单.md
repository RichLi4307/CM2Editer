# CM2Editer 问题清单

> 生成时间：2026-07-07 14:34
> 说明：本清单由旧《紧急修复清单》归档后重构，仅保留当前未解决问题。
> 优先级：P0（阻塞可用性）→ P1（体验问题）→ P2（ polish / 待验证）

---

## P0：交互阻塞（功能不完整，影响主要 workflow）

### P0.1 框选视觉与风格错误

**症状**：
- 空白处左键拖拽框选时，画布不显示选择框矩形。
- Shift/Ctrl + 空白处左键拖拽当前为 cross 风格（相交即选中），但需求为 window 风格（完全包含才选中）。
- 在节点上按住 Shift/Ctrl 再拖拽，仍然进入单节点拖动，而不是以该节点为起点开始框选。

**根因**：`src/ui/interaction.rs` 的 `handle_box_selecting` 使用 `Rect::intersects` 判定；状态机未绘制选择框；`handle_idle` 中节点命中优先于框选，未处理修饰键。

**涉及代码**：`src/ui/interaction.rs`

**期望行为**：
- 空白处拖拽：显示半透明选择框矩形。
- Shift + 空白处拖拽：window 风格加选。
- Ctrl + 空白处拖拽：window 风格减选。
- 在节点上按住 Shift/Ctrl 拖拽：以鼠标按下点为起点开始框选，不触发节点拖动。

---

### P0.2 多选节点无法整体拖拽

**症状**：框选多个节点后，只能拖动鼠标按住的那个节点，其他被选节点不会跟随移动。

**根因**：`handle_dragging_node` 仅移动 `state` 中记录的单节点；未遍历 `selected_nodes` 同步偏移。

**涉及代码**：`src/ui/interaction.rs`

**期望行为**：框选多个节点后，拖动其中任意一个，所有被选节点按相同世界偏移同步移动；释放后仅生成一条 `MoveNode` 命令或按节点生成多条命令入栈。

---

### P0.3 连线无法选中/删除

**症状**：
- 鼠标悬停或单击连线无法将其选中。
- 框选（cross/window）无法选中连线。
- 选中连线后按 Delete 无反应。

**根因**：`EdgeRenderer` 仅负责绘制，未返回命中区域；`interaction.rs` 没有针对连线的点击/框选检测逻辑；`selected_edges` 仅在添加边时被清空，从未被填充。

**涉及代码**：`src/ui/edge_renderer.rs`、`src/ui/interaction.rs`、`src/app.rs` 的 `delete_selected`

**期望行为**：
- 单击连线可选中（高亮）。
- 框选可包含连线（cross 或 window 按规则判定）。
- 选中连线后按 Delete 删除该边并同步更新 JSON 预览。

> 附带建议：将连线由贝塞尔曲线改为电路连接线风格（直角/折线），并在文档中同步该设计决定。

---

### P0.4 端口高亮/吸附与不兼容提示缺失

**症状**：
- 从输出端口拖线时，兼容输入端口不会高亮，也没有吸附效果。
- 拖到不兼容端口时不会变红，也没有视觉拒绝提示，仅松手后状态栏提示“目标端口不兼容或无效”。
- 端口高亮/吸附概率失败，缺少状态机可视化手段以便进一步排查。

**根因**：`handle_drawing_edge` 中仅读取 `end_pos` 用于释放时检测，未在拖拽过程中实时高亮兼容端口；未根据 `PortType::is_compatible_with` 反馈颜色。

**涉及代码**：`src/ui/interaction.rs` 的 `handle_drawing_edge`

**期望行为**：
- 拖拽连线时，鼠标附近的兼容输入端口高亮或放大。
- 悬停不兼容端口时，端口或临时连线变红，提示拒绝连接。
- 释放时仍保持现有“不创建悬空边”行为。

---

## P1：右键菜单与节点操作

### P1.1 右键菜单复制/粘贴未落地

**症状**：
- 右键节点菜单中的“复制”仅选中该节点，并未复制到剪贴板。
- 复制后没有“粘贴”选项。
- Ctrl+V 无法粘贴节点。

**根因**：`App::clipboard` 字段已存在，但 `show_context_menu` 中的“复制”未实现序列化/写入 `clipboard`；全局快捷键未处理 Ctrl+V；缺少粘贴位置计算逻辑。

**涉及代码**：`src/ui/interaction.rs` 的 `show_context_menu`、`src/app.rs` 的快捷键处理与 `clipboard`

**期望行为**：
- 右键“复制”将节点（含参数）写入 `clipboard`。
- 复制后在右键菜单中显示“粘贴”。
- 支持 Ctrl+V 在鼠标悬停位置创建副本；副本 ID 与源节点不冲突。

---

### P1.2 右键删除节点未级联删除边

**症状**：右键节点 → 删除，仅删除该节点，其关联边仍残留在图中和 JSON 里。

**根因**：右键菜单的“删除”使用 `Command::RemoveNode { node, edges: Vec::new() }`，未收集被级联删除的边。

**涉及代码**：`src/ui/interaction.rs` 的 `show_context_menu`、`src/app.rs` 的 `delete_selected`

**期望行为**：右键删除节点与 `delete_selected` 保持一致，删除节点时同时级联删除其输入/输出边，并支持撤销时恢复这些边。

---

## P2：文件与数据一致性

### P2.1 加载 JSON / 导出 `.code` 未接入 GUI

**症状**：
- 顶部工具栏只有“保存 (Ctrl+S)”“导出 JSON”“导出 .code”按钮。
- 没有“打开/加载 .json”按钮或菜单。
- 导出 `.code` 文件已生成，但尚未确认与节点逻辑完全对应。

**涉及代码**：`src/app.rs` 的工具栏、`src/serializer/json.rs`、`src/code_gen/generator.rs`

**期望行为**：
- 工具栏添加“打开 JSON”按钮，支持选择 `.json` 文件并加载到当前画布。
- 加载后节点位置、连线、参数与保存时完全一致。
- 确认保存的 JSON `version` 为 `"1.3"`。
- 导出 `.code` 后可用 `docs/examples/` 中的预期输出对照验证。

---

### P2.2 空画布/欢迎页未处理

**症状**：程序启动后默认加载 3 个示例节点（Start → If → Log），无空画布状态或欢迎页。

**涉及代码**：`src/app.rs` 的 `build_sample_graph`

**期望行为**：可配置启动为空画布（或保留示例图但提供清空/新建按钮）；空画布下添加第一个节点不 panic。

---

## P3：极端场景与验证器可视化

### P3.1 环检测无画布提示

**症状**：故意连接成环（A→B→A）时，底部状态栏显示错误文字，但画布区域无任何高亮或提示，且底栏错误无法展开详情。

**根因**：`App::validate` 仅更新 `validation_errors`，未将错误信息映射到节点/边渲染层，也没有错误详情弹窗。

**涉及代码**：`src/app.rs` 的 `validate`、`src/ui/node_renderer.rs`、`src/ui/edge_renderer.rs`、状态栏面板

**期望行为**：
- 环中涉及的节点/边高亮标红。
- 状态栏错误可点击展开详情列表。

---

### P3.2 复制粘贴性能未验证

**症状**：未测试复制粘贴 100 个节点后帧率是否保持 60fps。

**涉及代码**：整体复制粘贴实现

**期望行为**：复制粘贴大量节点后 `cargo run` 不卡顿，帧率稳定。

---

### P3.3 损坏 JSON 加载未处理

**症状**：加载损坏或版本不支持的 JSON 时，程序可能 panic 或无任何友好提示。

**涉及代码**：`src/serializer/json.rs`、`src/serializer/migration.rs`

**期望行为**：加载失败时弹出错误提示，说明失败原因，程序不 panic。

---

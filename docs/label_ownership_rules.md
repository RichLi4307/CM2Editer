# 标签管理规则（v0.2.2）

> 更新时间：2026-07-10
> 替代文档：`docs/label_management_plan.md`

## 标签创建方式

| 方式 | 触发条件 | graph.labels 变化 | 说明 |
|------|---------|-------------------|------|
| 左栏 `+ 新建标签` | 点击按钮 | `labels.insert("new_label", [])` | 空标签，需手动拖入 Label 节点或绑定节点 |
| 拖拽 Label 节点到画布 | 创建节点 | 无变化（需用户在属性面板填写 `name`）| |
| 属性面板修改 `name` | 失焦/回车 | 自动注册：`labels.entry(name).push(node_id)` + 清理旧归属 | ✅ 完整处理 |
| Goto / CreateThread 改参数 | 失焦/回车 | `labels.entry(target).or_default()` | 空标签，待绑定 Label 节点 |
| Data 连线 → Label.name | 连线建立 | ⚠️ 未处理（仅代码生成器感知） | 待修复 |

## 标签删除方式

| 方式 | 触发条件 | graph.labels 清理 |
|------|---------|-------------------|
| 左栏 `Del` 按钮 | 点击按钮 | ✅ 直接 `labels.remove(name)` |
| 删除节点 | Delete 键或右键删除 | ✅ `delete_selected` 自动清理孤立条目 |
| Label 改名 | 属性面板改 `name` | ✅ 旧名清理 + 空条目移除 |

## 所有权规则

1. **一个 Label 节点绑定到一个标签**：Label 的 `name` 参数值即标签名
2. **Label 改名 = 解绑旧标签 + 绑定新标签**：旧标签若不再有其他节点则自动清除
3. **节点删除 = 自动孤立检查**：标签下所有节点被删除时，标签自动清除
4. **Goto/CreateThread 目标是弱绑定**：仅创建空标签入口，不拥有节点。Label 节点才是强绑定（节点 ID 写入标签列表）
5. **main 标签不可删除/重命名**：界面已阻止操作

## 待解决的问题

| 问题 | 影响 | 优先级 |
|------|------|--------|
| Data 连线绑定未触发自动注册 | Label 节点连接 Data 边后名称不显示在标签管理器 | P1 |
| 多个同名 Label 节点 | 后创建的覆盖前者，无警告 | P2 |
| `+ 新建标签` 只创建空标签 | 用户需额外拖拽 Label 节点 | P3（UX） |

API

概述

文件结构

  • 每个项目在 "CustomMissions2" 文件夹内都有自己的文件夹，而该文件夹位于游戏的根目录中。

代码

  • 一个项目可以拆分成多个以 ".code" 结尾的文件。
  • 所有代码文件在加载时会被合并成一个文件。
    合并顺序可能是按字母顺序，但你不应依赖这一点，而应确保它们在任意顺序下都能正常工作。
  • 编码应为 UTF-8，以确保字符串字符能正确导入游戏。

Meta（元数据）

  • 你可以在项目中包含一个 "meta.json" 文件，用来指定不同语言下的标题、添加描述，并构建一个简单的设置菜单。
  • 设置值可以在代码中通过 _settings 全局变量读取。
{
	"title":{
		"En":"Test",
		"Ja":"テスト"
	},
	"description":{
		"En":"This is a description"
	},
	"settings":[
		{
			"type":"Label",
			"title":"These settings don't do anything"
		},
		{
			"name":"range",
			"title":"Range Integer",
			"type":"Integer",
			"minvalue":0,
			"maxvalue":100,
			"default":50
		},
		{
			"name":"range2",
			"title":"Range Float",
			"type":"Float",
			"minvalue":0,
			"maxvalue":5,
			"default":5
		},
		{
			"name":"checked",
			"title":"Boolean Value",
			"type":"Boolean",
			"default":true
		},
		{
			"name":"text",
			"title":"Text Input",
			"type":"String",
			"default":""
		},
		{
			"name":"enum",
			"title":"Options",
			"type":"Enum",
			"options":[
				"Option 1",
				"Option 2",
				"Option 3"
			],
			"default":0
		}
	]
}
  • title、description 以及 Enum 的选项，必须是普通字符串，或者是以语言代码命名各元素的对象。（另见 GetLanguage）
  • 如果你没有为某种语言指定翻译，它会默认使用列表中的第一项。（示例中为 "En"）
  • 这一点对设置项同样适用。
  • settings 是一个可选的对象数组。
  • 设置项的 name 决定了你在代码中通过全局 _settings 列表访问它时所用的名称。（例如 "_settings.range"）
  • 可用的设置类型有：Label、Integer、Float、Boolean、String、Enum
  • Integer 和 Float 会创建一个滑块，因此你还应指定 minvalue 和 maxvalue
  • Enum 会创建一个选择器，你可以在各选项间循环切换，但 _settings 中的值将是一个数字
  • Boolean 会创建一个复选框，String 则创建一个单行文本输入框。
  • Label 只是给设置菜单添加一段说明文字，因此它不需要 name。
  • default 以对应的类型指定初始值。
  • 设置会随每个存档保存，在游戏存档时写入。

内置全局变量

  • _state
    • 包含大量值的列表，如条件状态、位置、任务、物品和角色扮演服装。参见 附录
    • 不要用它来修改值，而应使用现有的函数
    • 该列表仅在每帧之前更新。因此当你以其他方式修改某个值时，它不会立即反映在列表中。
  • _stagechanged
    • 本帧与上一帧之间场景是否发生了变化
  • _timediff
    • 自上一帧以来经过的游戏内时间
    • 会考虑时间被减速或暂停的情况
  • _time
    • 累计时间，已计入减速和暂停
  • _save
    • 读写该列表中的项目，以创建持久化的存档状态。
    • 对象（List 除外）无法被保存
  • _settings
    • 读取你可以通过智能手机图形界面修改的设置的值。
    • 关于如何设置菜单，请参见 Meta。
  • _mod
    • 一个可以填入数据的列表，其数据可被其他项目访问。
    • 不允许存放对象（List 除外），它们会被移除。
    • 关于如何从另一个项目访问这些数据，请参见 "_mods"。
  • _mods
    • 所有已激活项目的列表。
    • 以数字为索引的项目以字符串形式包含项目名称（文件夹名），因此你可以遍历所有已激活的项目。
    • 以项目名称为索引的项目是列表，包含相应项目通过 _mod 也能访问到的数据。
    • 对象（List 除外）无法被共享，会被移除。
  • _name
    • 项目自身的名称，与它在 _mods 中使用的名称相同。

控制结构

代码缩进

  • 代码缩进用于确定代码的结构。
  • 在标签（label）或 If 语句之后的行使用单个制表符（Tab），以表示该代码属于它们。

标签（labels）

  • 标签用于创建自定义函数，以及作为线程的入口点。
  • 标签名称之后跟一个冒号（":"）。
  • 与 Python 不同，If 语句和 While 循环不使用冒号。
  • 标签可以嵌套。
Log(welcome_msg(name = "Player"))

welcome_msg:
	_result = "Welcome "+name

变量作用域

  • 函数和线程会为局部变量创建各自的作用域。
  • 函数和线程可以访问在更高作用域中定义的变量。
  • 如果你向函数或线程传递一个带名称的参数，它会隐藏更高作用域中同名的变量。
  • 使用同一标签的多个线程会创建各自的变量实例，但共享其父作用域中的变量。
name = "Player"
function1(id=1)
function1(id=2)

function1:
	sqr = id*id
	Log(name+id+": sqr="+sqr)

输出：
Player1: sqr=1
Player2: sqr=4

if、elseif、else

if x == 0
	Log("zero")
elseif x < 0
	Log("less than zero")
else
	Log("greater than zero")

while

  • 你可以使用 break 语句提前退出循环。
Log("count down")
i = 10
while i>=0
	Log(i)
	i = i-1

for

  • 遍历列表中的各个值。
  • 若要遍历一个数字范围，你可以使用 Range 函数创建一个临时列表。
  • 你可以使用 break 语句提前退出循环。
for i in Range(5,10)
	Log(i)

输出：
5
6
7
8
9

函数调用

  • 函数通过标签（label）来定义。
  • 调用形式为：labelname(Parameter1, Parameter2, ...)
  • 参数可以是具名的，也可以是匿名的。
  • 匿名参数通过列表 _args 访问。
  • 函数返回变量 _result 的值（若未设置，则 _result 为 null）。
  • 你可以使用 return 语句提前退出函数。如果在 return 后跟一个表达式，它会隐式地将 _result 设为该值。
variable1 = function_add(2, 3)
Log(variable1) Output: 5
variable2 = function_mult(a=2, b=3)
Log(variable2) Output: 6

function_add:
	_result = _args[0]+_args[1]

function_mult:
	return a * b

Constructors

CreateCondition(Condition[, id = ID])
返回：Object
  • 条件使用简短而简单的语法，来创建对玩家状态更复杂的组合，这些状态也可以通过 _state 全局变量访问。

  • 要用逻辑与（AND）组合多个条件类型，请用方括号包住以逗号分隔的列表。

创建一个条件：你必须暴露正面和背面，但不能完全裸体。
方括号表示所有条件都必须为真，而感叹号对 Exposed_All 取反，因此它必须为假才能满足整体要求。
condition1 = CreateCondition("[Exposed_Front,Exposed_Hip,!Exposed_All]")

创建一个条件：你必须裸体、使用已启动的阴蒂跳蛋，并对壁挂式假阳具执行一个动作。
condition2 = CreateCondition("[Exposed_All,AdultToy_KuriRotor,(VibrationLow,VibrationHigh,VibrationRandom),(Action_UseDildoWallPussy1,Action_UseDildoWallAnal1,Action_UseDildoWallFella1)]")

  • 某些条件可以用运算符 >、>=、<、<=、==、!= 与数字进行比较。

  • 你还可以在创建条件时设置一个 ID，并在另一个条件中通过条件类型 "Subcondition" 将其用作子条件。
  • 这对 ItemCondition 同样适用。
  • 只要条件比对 _state 变量中单个值的简单比较更复杂，你就应尽量使用预先准备好的条件。

Delay:	
	if delaymanager == null
		Global("delaymanager", CreateThread("delaymanager"))
	if GetType(_args[0]) == "Object"
		delaymanager.AddMethod(thread = _args[0], method = _args[1], delay = _args[2])
	else
		delaymanager.AddMethod(thread = null, method = _args[0], delay = _args[1])

delaymanager:
	listeners = CreateList()
	
	AddMethod:
		index = 0
		while listeners[index] != null
			index+=1
	
		newlistener = CreateListener("ListenerMethods", thread = thread, method = method, delay = delay, index = index)
		listeners[index] = newlistener
		
	ListenerMethods:
		delay -= _timediff
		if delay <= 0
			if thread != null
				CallMethod(thread, method)
			else
				CallFunction(method)
			listeners[index] = null

    • 角色扮演服装的名称是列表的键。

  • Missions
    • 按任务 ID 记录该任务的通关次数。
    • 还包含 "Completed" 和 "Count"，分别表示已通关的不同任务数量和现有任务的总数。
    • 还为每个舞台提供单独的列表。 
      stage = "Residence"
Log(Format("Cleared Missions in \"{0}\": {1}/{2}", stage, _state.Missions[stage].Completed, _state.Missions[stage].Count))
  • CurrMissions
    • 按任务 ID 记录该任务的当前进度（0 到 1）。

    • 还为每个舞台提供单独的列表。

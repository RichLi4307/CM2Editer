## P0：代码生成器全面重构（进行中）

### 已修正

| 问题 | 修复 |
|------|------|
| `main:` 被当作硬编码入口 | 顶层 `CreateThread("main")` 才是真正的触发 |
| 所有代码塞在 `main:` 下 | 顶层代码 + 标签体分离 |

### 待修正

| 问题 | 说明 |
|------|------|
| **`_result = null` 缺失** | 每个标签结尾应添加，否则 GoTo 后状态残留 |
| **多 Start → 多入口** | 仅对 `main*` 标签生成顶层 CreateThread；多线程模型需要支持任意标签 |
| **Listener 标签** | `CreateListener("label")` 应每帧执行，当前生成错误 |
| **Goto 参数传递** | `thread.Goto("label", data=obj, nextstep="x")` 需要生成完整的键值对 |
| **函数调用标签** | `labelName(args)` 直接调用模式完全未支持 |
| **Foreach 节点** | 需要 `i = Foreach(list, threadVar)` 生成 |
| **跨线程 GoTo** | `main.Goto("label")` 模式 |

### 验证步骤

1. 打开现有工程 → 生成 `.code` → 对比 `docs/code_pseudocode_map.md` 中同结构文件的伪代码
2. 测试多线程模型：一个 Start 分叉出 2 个 `CreateThread` → 每个线程单独标签
3. 测试 `thread.Goto` + 参数 → 生成的 `.code` 能被 CM2 加载器解析
4. 测试 Listener 模式 → `CreateListener("x")` → `x:` 标签每帧运行

## P1：代码生成逻辑链路

| 任务 | 说明 |
|------|------|
| **标签生命周期** | 每个标签 → Goto 跳入 → 执行到底 → Goto 跳出。需正确处理 `_result = null` 收尾 |
| **线程作用域** | 同一线程内标签共享变量；Goto 清除旧上下文。需在生成时保留线程局部变量 |
| **参数传递语法** | `thread.Goto("x", key=value)` → key 成为目标标签的局部变量 |
| **Listener 作用域** | `CreateListenerLocal("x")` vs `CreateListener("x")` 作用域不同——Local 保留调用者的局部变量 |

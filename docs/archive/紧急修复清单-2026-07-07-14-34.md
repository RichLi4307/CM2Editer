## P0：输入精度（数学错误，必须修）

### P0.1 滚轮缩放跳跃（一滚跳 0.5x）

**症状**：Windows 滚轮设置"每次滚动 N 行"，egui 把 N 行累积成一个 `scroll_delta`，导致一格滚轮触发多次缩放，倍率乱跳。

**根因**：`src/ui/canvas.rs` 第 98 行 `zoom_at` 使用 `scroll * 0.1`，缩放增量与滚轮 delta 大小成正比，受 Windows 滚轮灵敏度影响。

**实际代码位置**：`src/ui/canvas.rs`

```rust
// 当前代码（错误）
fn zoom_at(&mut self, screen_pos: Pos2, canvas_rect: Rect, scroll: f32) {
    let old_zoom = self.viewport.zoom;
    let new_zoom = (old_zoom * (1.0 + scroll * 0.1)).clamp(0.1, 4.0);
    ...
}
```

**修复方案**：按事件方向处理，只认正负，不认大小。一格放大 10%，一格缩小 10%。

```rust
// 替换为：
fn zoom_at(&mut self, screen_pos: Pos2, canvas_rect: Rect, scroll: f32) {
    if scroll == 0.0 {
        return;
    }
    let old_zoom = self.viewport.zoom;
    let new_zoom = if scroll > 0.0 {
        (old_zoom * 1.1).clamp(0.1, 4.0)
    } else {
        (old_zoom / 1.1).clamp(0.1, 4.0)
    };
    if (new_zoom - old_zoom).abs() < f32::EPSILON {
        return;
    }
    let m = screen_pos - canvas_rect.center();
    let factor = 1.0 / old_zoom - 1.0 / new_zoom;
    self.viewport.x += m.x * factor;
    self.viewport.y += m.y * factor;
    self.viewport.zoom = new_zoom;
}
```

> 注意：使用 `raw_scroll_delta.y` 判断方向。关键是**判断正负，不乘具体数值**。

**验证**：

- [pass] 从 1.0x 滚一格 → 1.1x
- [pass] 再滚一格 → 1.21x
- [pass] 反向滚一格 → 1.1x
- [pass] 快速滚 10 格和慢速滚 10 格结果一致（无累积漂移）

### P0.2 中键平移滑过头（画布比鼠标跑得快）

**症状**：按住中键拖拽，画布移动距离 > 鼠标移动距离，且 zoom 越大滑得越夸张。

**根因**：`src/ui/canvas.rs` 和 `src/ui/interaction.rs` 同时处理了中键拖拽，每帧平移两次。实际单个实现里的公式 `viewport -= delta / zoom` 是正确的，但两处重复执行导致速度翻倍。在 zoom 较大时，虽然单次 `delta / zoom` 变小，但两次累加后仍会让节点明显跟不上鼠标。

**实际代码位置**：

- `src/ui/canvas.rs` 第 64–68 行（中键平移）
- `src/ui/interaction.rs` 第 115–117 行、第 268–288 行（`Panning` 状态处理）

**修复方案**：只保留一处处理。`Canvas` 负责视口操作，删除 `interaction.rs` 中的 `Panning` 状态与相关处理，避免重复。

```rust

```

// 2. handle_idle 中处理中键拖拽的逻辑
// 3. handle_panning 方法

## P1：渲染与显示（体验问题）

### P1.1 中文显示方块（□□□）

**症状**：所有中文字符显示为 □，英文正常。

**根因**：egui 默认只嵌入 ASCII 字体，不含 CJK 字形。`src/app.rs` 当前没有加载任何中文字体。

**实际代码位置**：`src/app.rs`（应用构造阶段）和 `src/main.rs`（`eframe::run_native` 的创建闭包）

**修复方案**：在 `eframe::CreationContext` 中加载 Windows 系统字体（微软雅黑）作为 fallback，无需随仓库分发字体文件。

> RichLi: 要不得要不得，我下载思源黑体放docs目录了，你拿去移动到该放的位置，就用这个

```rust
// src/app.rs
pub fn new(cc: &eframe::CreationContext) -> Self {
    setup_fonts(&cc.egui_ctx);
    ...
}

fn setup_fonts(ctx: &egui::Context) {
    let mut fonts = egui::FontDefinitions::default();

    #[cfg(target_os = "windows")]
    {
        let candidates = [
            r"C:\Windows\Fonts\msyh.ttc",
            r"C:\Windows\Fonts\msyhbd.ttc",
            r"C:\Windows\Fonts\simhei.ttf",
        ];
        for path in &candidates {
            if let Ok(bytes) = std::fs::read(path) {
                fonts.font_data.insert(
                    "cjk".to_owned(),
                    egui::FontData::from_owned(bytes),
                );
                fonts
                    .families
                    .entry(egui::FontFamily::Proportional)
                    .or_default()
                    .push("cjk".to_owned());
                fonts
                    .families
                    .entry(egui::FontFamily::Monospace)
                    .or_default()
                    .push("cjk".to_owned());
                break;
            }
        }
    }

    ctx.set_fonts(fonts);
}
```

```rust
// src/main.rs
Box::new(|cc| Ok(Box::new(App::new(cc))))
```

> 如果系统字体不存在，程序仍使用默认字体，不会 panic。

### P1.2 左栏目录每帧打乱顺序

**症状**：左侧节点库（Node Library）的分类列表每帧随机重排。

**根因**：`src/ui/panels/node_library.rs` 第 20 行使用 `HashMap` 按分类聚合节点定义，`HashMap` 迭代顺序不稳定，每帧重绘都不同。

**实际代码位置**：`src/ui/panels/node_library.rs`

```rust
// 当前代码（错误）
let mut categories: std::collections::HashMap<
    &str,
    Vec<&crate::api::definitions::NodeDefinition>,
> = std::collections::HashMap::new();
for def in defs {
    categories.entry(&def.category).or_default().push(def);
}
```

**修复方案**：将 `HashMap` 替换为 `BTreeMap`，按键自动排序。

```rust
// 替换为：
let mut categories: std::collections::BTreeMap<
    &str,
    Vec<&crate::api::definitions::NodeDefinition>,
> = std::collections::BTreeMap::new();
for def in defs {
    categories.entry(&def.category).or_default().push(def);
}
```

**验证**：

- [pass] 左栏分类顺序固定（如 Always Control → File → Game Functions...）
- [pass] 切换节点、缩放、拖拽后顺序不变
